Footnotes

* Disk Size (disk_size) — Total disk file size consumed by the index. This is the compressed on-disk size and includes storage overhead, fragmentation, and metadata.

** Data Size (data_size) — The size of indexable data maintained for the index. This is the uncompressed logical size. It is normal for Data Size to be larger than Disk Size because Couchbase's Plasma storage engine compresses index data on disk.

Bloat (disk ÷ data) — When < 1.0x, on-disk compression is working well. When > 1.0x, overhead from fragmentation or metadata exceeds savings from compression.
